# 股票形态相似度匹配与机器学习训练工程操作说明

## 1. 工程定位

这个工程主要用于两类事情：

1. **形态相似度匹配**
   
   你提供一只或几只已经走出来的强势股，工程会提取它们的启动前或启动期形态，然后去 `cache/hist` 股票池中寻找当前走势相似的股票。

2. **机器学习模型训练**
   
   使用历史日线窗口构建训练样本，用随机森林模型学习“过去一段走势形态”和“未来是否上涨”之间的关系，训练完成后生成模型文件，后续可以用于打分或回测。

需要注意：

```text
ml_match.py      = 手动指定股票 + 时间段，做模板匹配
ml_similarity.py = 只给股票代码，自动提取模板，做相似度匹配
ml_train.py      = 训练机器学习模型
ml_backtest.py   = 加载训练好的模型做历史回测
```

---

# 4. ml_match.py 使用说明

## 4.1 功能说明

`ml_match.py` 用于：

```text
手动指定一只股票 + 一段强势区间
↓
提取这段区间对应的形态模板
↓
扫描 cache/hist 股票池当前最近走势
↓
输出相似度最高的股票
```

适合你已经明确知道某只股票哪一段走势是经典启动段。

---

## 4.2 基础命令

例如使用 `002179` 在 `2026-04-20 ~ 2026-05-10` 的走势作为模板：

```bash
python cli/ml_match.py --code 002179 --date-start 2026-04-20 --date-end 2026-05-10 --threshold 0.55 --top-k 20
```

含义：

```text
--code 002179              模板股票代码
--date-start 2026-04-20    模板区间开始日期
--date-end 2026-05-10      模板区间结束日期
--threshold 0.55           相似度阈值，低于55%的匹配丢弃
--top-k 20                 输出相似度最高的前20只股票
```

---

## 4.3 快速只看相似股票，不跑回测

推荐你日常先用这个：

```bash
python cli/ml_match.py --code 002179 --date-start 2026-04-20 --date-end 2026-05-10 --threshold 0.55 --top-k 20 --recent-windows 1 --skip-backtest --workers 8
```

参数解释：

```text
--recent-windows 1    每只候选股只取当前最新1个窗口
--skip-backtest       跳过历史回测，只输出当前相似股票
--workers 8           使用8线程扫描候选股票
```

这个命令速度最快，适合盘后快速筛选。

---

## 4.4 输出结果怎么理解

示例：

```text
002265 建设工业: 平均67.0% 最大71.0% 匹配4次
```

含义：

```text
平均67.0%  = 超过阈值的匹配结果平均相似度
最大71.0%  = 和某一个模板窗口的最高相似度
匹配4次    = 候选股当前窗口和多个模板窗口比较后，有4次超过阈值
```

如果某只股票：

```text
最大相似度高
匹配次数多
```

说明它不是偶然和某一天相似，而是和模板股的一段连续形态都比较接近。

---

## 4.5 注意事项

当前 `ml_match.py` 的匹配逻辑是：

```text
002179 在 2026-04-20 ~ 2026-05-10 的历史模板
vs
候选股票当前最新20日窗口
```

不是比较：

```text
002179 2026-04-20 ~ 2026-05-10
vs
候选股 2026-04-20 ~ 2026-05-10
```

也就是说，它是找“现在像不像那段历史强势形态”，不是做同一时间段 PK。

# 5. ml_similarity.py 使用说明

## 5.1 功能说明

`ml_similarity.py` 用于：

```text
只给股票代码，不给时间段
↓
工程自动从这些股票中提取启动前或启动期模板
↓
扫描 cache/hist 股票池当前形态
↓
输出当前相似股票
```

适合你只知道某几只股票最近比较强，但不想手动选择强势区间。

---

## 5.2 单只股票自动提取模板

```bash
python cli/ml_similarity.py --template 002179 --template-mode auto --threshold 0.55 --top-k 20 --recent-windows 1 --workers 8
```

含义：

```text
--template 002179       使用002179作为模板股票
--template-mode auto    自动识别启动点并提取模板窗口
--threshold 0.55        相似度阈值
--top-k 20              输出前20只
--recent-windows 1      候选股只取最新窗口
--workers 8             并发扫描
```

---

## 5.3 多只股票自动提取模板

```bash
python cli/ml_similarity.py --template 002179,600535,002265 --template-mode auto --threshold 0.55 --top-k 30 --recent-windows 1 --workers 8
```

这表示：

```text
从 002179、600535、002265 这几只股票里自动提取强势形态模板
然后去 cache/hist 股票池中找当前走势相似的股票
```

多只股票会形成一个模板库，通常比单只股票更稳。

---

## 5.4 template-mode 参数说明

```text
auto       自动判断启动点，推荐使用
prelaunch  提取启动前窗口
launch     提取启动期窗口
both       启动前 + 启动期都提取
recent     直接提取最近窗口
all        提取历史所有窗口，不推荐日常使用
```

推荐顺序：

```text
优先使用 auto
如果你想找启动前蓄势形态，用 prelaunch
如果你想找正在启动的形态，用 launch 或 both
不要轻易使用 all，因为会把下跌、震荡阶段也当成模板
```

---

## 5.5 ml_similarity.py 和 ml_match.py 的区别

```text
ml_match.py：你告诉代码哪一段是强势期
ml_similarity.py：你只告诉代码哪几只是强势股，让代码自动找模板
```

如果你很确定强势区间，优先用 `ml_match.py`。

如果你不想手动选日期，使用 `ml_similarity.py`。

# 6. ml_train.py 使用说明

## 6.1 功能说明

`ml_train.py` 用于训练机器学习模型。

它和相似度匹配不同。

相似度匹配是：

```text
模板窗口 vs 候选窗口
看两个走势形态像不像
```

机器学习训练是：

```text
历史20日走势窗口 = X
未来N日是否上涨达到目标收益 = y
训练模型学习 X 和 y 的关系
```

训练完成后会生成模型文件，例如：

```text
output/ml_models/ml_pattern_model.pkl
```

后续可以用这个模型对股票当前形态打分，或者做历史回测。

---

## 6.2 基础训练命令

使用指定模板股票训练：

```bash
python cli/ml_train.py --template 002179,600535,002265 --validate
```

含义：

```text
--template 002179,600535,002265   以这些股票作为重点样本来源
--validate                        训练完成后输出验证指标
```

训练完成后，通常会输出：

```text
accuracy
precision
recall
f1
auc_roc
```

模型会保存到：

```text
output/ml_models/ml_pattern_model.pkl
```

---

## 6.3 自动选择强势股训练

如果你不想手动指定模板股票，可以让工程自动挑选近期强势股：

```bash
python cli/ml_train.py --auto-template --auto-top-n 20 --validate
```

含义：

```text
--auto-template     自动从股票池中选择近期强势股
--auto-top-n 20     选择最近涨幅较强的前20只
--validate          输出验证结果
```

这个适合快速测试，但不一定最符合你的交易审美。

更推荐你手动给一批你认可的强势股。

---

## 6.4 推荐训练方式

推荐先准备一批你认为走势质量比较好的强势股，例如：

```text
002179
600535
002265
000988
600498
```

然后运行：

```bash
python cli/ml_train.py --template 002179,600535,002265,000988,600498 --validate
```

如果样本太少，工程一般会自动从其他股票补充样本。

---

## 6.5 训练目标是什么

默认训练目标通常类似：

```text
过去 lookback 个交易日的形态
预测未来 hold_days 个交易日内收益是否达到 target_pct
```

通俗理解：

```text
过去20天走势长这样
未来5天是否能涨超过5%
```

如果满足目标，样本标签为：

```text
label = 1
```

否则：

```text
label = 0
```

---

## 6.6 常见参数说明

不同版本参数可能略有差异，你可以用下面命令查看：

```bash
python cli/ml_train.py -h
```

常见参数包括：

```text
--template          手动指定模板股票列表，逗号分隔
--auto-template     自动选择近期强势股作为模板
--auto-top-n        自动选择多少只强势股
--lookback          每个训练样本使用多少个交易日窗口，默认一般为20
--validate          训练后输出验证指标
--full-pipeline     训练后继续执行相似度扫描和回测
--threshold         后续扫描或模型判断阈值
```

如果你的版本支持 `--hold-days` 或 `--target-pct`，则可以控制预测目标。

例如：

```bash
python cli/ml_train.py --template 002179,600535,002265 --validate --hold-days 5 --target-pct 0.05
```

表示训练目标是：

```text
未来5个交易日收益是否达到5%
```

如果你的当前脚本提示 `unrecognized arguments`，说明该版本暂时没有这个参数，需要在 `ml_train.py` 中补充 argparse 参数。

---

## 6.7 训练完成后怎么用

训练完成后会生成模型：

```text
output/ml_models/ml_pattern_model.pkl
```

之后可以用 `ml_backtest.py` 做回测：

```bash
python cli/ml_backtest.py --model output/ml_models/ml_pattern_model.pkl --hold-days 3,5,10 --threshold 0.65
```

也可以在代码里加载模型，对股票当前走势打分。

典型流程是：

```text
ml_train.py 训练模型
↓
生成 ml_pattern_model.pkl
↓
ml_backtest.py 验证历史表现
↓
ml_strategy.py 或其他扫描逻辑用于当前股票打分
```

---

## 6.8 ml_train.py 的局限

当前训练模型需要注意几个问题：

1. **训练标签比较粗**

   例如“未来5天涨5%”只是一个简单定义，不一定等于你真实想要的主升行情。

2. **随机切分可能高估效果**

   同一只股票相邻日期的窗口非常相似，如果训练集和验证集随机切分，模型可能看起来效果不错，但实际泛化能力一般。

3. **没有充分考虑真实交易限制**

   例如涨停买不进、跌停卖不出、滑点、手续费、停牌等。

4. **样本质量决定模型质量**

   如果模板股票本身选得不好，模型会学偏。

因此，当前更推荐你先把 `ml_train.py` 作为辅助验证工具，不要直接把模型输出当成买入信号。

# 7. ml_backtest.py 使用说明

## 7.1 功能说明

`ml_backtest.py` 用于加载已经训练好的模型，扫描历史数据并统计模型信号表现。

基础命令：

```bash
python cli/ml_backtest.py --model output/ml_models/ml_pattern_model.pkl
```

指定持有周期：

```bash
python cli/ml_backtest.py --model output/ml_models/ml_pattern_model.pkl --hold-days 1,3,5,10
```

调整模型信号阈值：

```bash
python cli/ml_backtest.py --model output/ml_models/ml_pattern_model.pkl --threshold 0.70
```

---

## 7.2 回测规则

通常回测逻辑为：

```text
T日出现模型信号
T+1日开盘买入
T+N日收盘卖出
统计收益率
```

这样设计是为了避免未来函数。

不要用：

```text
T日信号出现后，T日收盘买入
```

因为实际盘后运行时，你无法在当天收盘前知道完整信号。

# 8. 相似度是怎么计算的

当前相似度不是直接比较同花顺 PK 图中的两条涨跌幅曲线。

当前逻辑大致是：

```text
20日窗口
↓
提取均线、涨幅、量能、相对高低点等特征
↓
标准化
↓
余弦相似度
```

所以它比较的是：

```text
技术结构像不像
```

不完全等于：

```text
肉眼看涨跌幅曲线像不像
```

如果你想让结果更接近同花顺 PK 图，需要下一步增加：

```text
20日累计涨跌幅曲线相似度
```

并提高它的权重。

推荐未来改成：

```text
最终相似度 = 累计涨跌幅曲线相似度 × 0.6
           + 日涨跌节奏相似度 × 0.2
           + 技术结构相似度 × 0.2
```

这样会更符合肉眼判断。

# 9. 参数调节建议

## 9.1 threshold

```text
0.45  结果多，适合初筛
0.55  默认推荐
0.60  更严格
0.65  很严格，可能结果较少
```

日常建议从：

```bash
--threshold 0.55
```

开始。

---

## 9.2 top-k

```text
--top-k 20   输出前20只
--top-k 30   输出前30只
--top-k 50   输出前50只
```

如果你只是人工复盘，建议：

```bash
--top-k 20
```

---

## 9.3 recent-windows

```text
--recent-windows 1   只看当前最新窗口，最快
--recent-windows 3   看最近3个窗口，更稳但慢一点
--recent-windows 5   更宽松，容易匹配到更多股票
```

日常建议：

```bash
--recent-windows 1
```

如果想提高容错，可以用：

```bash
--recent-windows 3
```

---

## 9.4 workers

```text
--workers 4   稳定
--workers 8   推荐
--workers 16  不一定更快，可能受硬盘IO限制
```

Windows 本地读大量 CSV 时，并发太高不一定更快。

---

## 9.5 skip-backtest

如果只是想快速看当前相似股票：

```bash
--skip-backtest
```

如果想验证这个模板历史上是否有效，就不要加这个参数。

# 11. 常见问题

## 11.1 unrecognized arguments: --workers

说明当前脚本没有加这个参数。

需要检查：

```text
cli/ml_match.py
cli/ml_similarity.py
```

是否已经替换为支持 `--workers` 的版本。

---

## 11.2 unrecognized arguments: --skip-backtest

说明当前 `ml_match.py` 没有加跳过回测参数。

需要替换支持 `--skip-backtest` 的版本。

---

## 11.3 为什么结果看起来不像同花顺 PK 图

因为当前相似度主要比较综合技术结构，不是单纯比较累计涨跌幅曲线。

如果你想让结果更像肉眼看到的走势，需要增加“累计涨跌幅曲线相似度”。

---

## 11.4 为什么 002100 没有名称

说明该股票在缓存文件或名称映射中没有读到名称。

可以在结果输出处增加兜底：

```text
如果名称为空，从CSV最后一行名称字段读取
如果还为空，显示未知
```

# 12. 当前工程建议优化方向

下一步最值得做的是：

1. **增加累计涨跌幅曲线相似度**

   让结果更接近你肉眼看图的感觉。

2. **输出最相似模板编号和日期**

   例如：

   ```text
   最像模板: 模板8 2026-04-02 ~ 2026-04-30
   候选窗口: 2026-04-22 ~ 2026-05-22
   ```

3. **Top20 输出更多辅助指标**

   例如：

   ```text
   近5日涨幅
   近10日涨幅
   近20日涨幅
   距60日高点
   距60日低点
   量能比
   ```

4. **增加图表输出**

   对每个 Top20 股票画：

   ```text
   模板股累计涨幅曲线
   候选股累计涨幅曲线
   ```

   这样你可以一眼判断像不像。

5. **模型训练改成时间切分验证**

   避免随机切分导致验证结果虚高。

# 13. 最推荐的三个命令

## 13.1 手动指定强势段，快速找当前相似股

