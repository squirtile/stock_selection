# 股票形态相似度匹配工程操作说明

## 1. 这个工程现在在做什么

当前这套 `ml_engine` 工程的核心目标是：

> 给定一只或几只已经走出强势行情的股票，从它们的历史走势中提取形态模板，然后扫描 `cache/hist` 股票池，寻找当前最近走势与模板相似的股票。

它不是实时盘口监控，也不是资金流监控，而是基于本地日线缓存数据做 **日线形态相似度匹配**。

你现在的股票池就是：

```text
cache/hist/*.csv
```

每个文件类似：

```text
cache/hist/002179_bs.csv
cache/hist/600535_bs.csv
cache/hist/002265_bs.csv
```

每个 CSV 一般代表一只股票的历史日线数据。

---

## 2. 当前工程支持的两种主要用法

### 2.1 手动指定模板区间

这是你已经跑通的方式。

适合场景：

> 你明确知道某只股票哪一段是强势启动期，想拿这一段当模板，去找当前走势类似的股票。

命令示例：

```bash
python cli/ml_match.py --code 002179 --date-start 2026-04-20 --date-end 2026-05-10 --threshold 0.55 --top-k 20 --recent-windows 1 --skip-backtest --workers 8
```

含义是：

```text
用 002179 在 2026-04-20 到 2026-05-10 这段形态作为模板；
扫描 cache/hist 股票池；
每只候选股只取当前最新 1 个 20 日窗口；
相似度超过 55% 才保留；
输出相似度最高的前 20 只；
跳过历史回测；
使用 8 个线程扫描。
```

这是目前最推荐你先使用的方式。

---

### 2.2 只给股票代码，自动提取模板

适合场景：

> 你不想手动指定时间段，只想告诉代码“这只股票最近很强”，让代码自动判断启动点并提取模板。

命令示例：

```bash
python cli/ml_similarity.py --template 002179 --template-mode auto --threshold 0.55 --top-k 20 --recent-windows 1 --workers 8
```

多只股票模板：

```bash
python cli/ml_similarity.py --template 002179,600535,002265 --template-mode auto --threshold 0.55 --top-k 30 --recent-windows 1 --workers 8
```

含义是：

```text
给代码一只或几只强势股；
代码自动从这些股票中寻找启动前或启动初期窗口；
把这些窗口合成模板库；
再扫描 cache/hist 股票池；
寻找当前最近走势相似的股票。
```

注意：自动模板提取不一定完全准确。它适合快速测试，但如果你已经知道最经典的启动区间，优先用 `ml_match.py` 手动指定时间段。

## 3. 当前匹配逻辑到底是什么

以这个命令为例：

```bash
python cli/ml_match.py --code 002179 --date-start 2026-04-20 --date-end 2026-05-10 --threshold 0.55 --top-k 20 --recent-windows 1 --skip-backtest --workers 8
```

实际执行流程是：

```text
Step 1：提取 002179 模板形态
Step 2：扫描 cache/hist 股票池当前最近形态
Step 3：如果没有 --skip-backtest，则做历史回测
```

---

## 4. Step 1：模板是怎么提取的

你给的是：

```text
002179
2026-04-20 到 2026-05-10
```

代码不会只截取这一段裸 K 线，而是会以这段时间里的每个交易日作为窗口结束日，向前取 `lookback` 个交易日。

默认：

```text
lookback = 20
```

所以会生成类似：

```text
模板0: 2026-03-23 ~ 2026-04-20
模板1: 2026-03-24 ~ 2026-04-21
模板2: 2026-03-25 ~ 2026-04-22
...
模板11: 2026-04-08 ~ 2026-05-08
```

也就是说，你给的日期区间如果有 12 个有效交易日，就会生成 12 个模板窗口。

每个模板窗口代表：

```text
这只股票在某个模板结束日之前 20 个交易日的形态
```

---

## 5. Step 2：候选股票怎么匹配

当前候选池是：

```text
cache/hist
```

代码会遍历里面所有股票。

如果你的输出是：

```text
候选股票: 1155 只；候选窗口: 1142 个
```

意思是：

```text
cache/hist 里面一共有 1155 只股票；
其中 1142 只成功提取出当前最近 20 日形态窗口；
其余股票可能因为数据不足、字段缺失或指标 NaN 被跳过。
```

如果你用了：

```bash
--recent-windows 1
```

那么每只候选股只取：

```text
当前最新 1 个 20 日窗口
```

如果你改成：

```bash
--recent-windows 3
```

那么每只候选股会取最近 3 个窗口：

```text
最新窗口：最近 20 个交易日
前一窗口：往前错开 1 天的 20 日窗口
再前一窗口：往前错开 2 天的 20 日窗口
```

`recent-windows` 越大，匹配更充分，但速度会更慢，也可能引入更多噪声。

---

## 6. 当前相似度是怎么计算的

当前版本的相似度不是直接比较同花顺 PK 图那种累计涨幅曲线，而是比较综合技术特征。

每个 20 日窗口会被转成一组特征，大概包括：

```text
收盘价相对均线的位置
SMA5 / SMA20
SMA10 / SMA20
SMA20 / SMA60
当前价距离 60 日高点的位置
当前价距离 60 日低点的位置
近 5 日涨幅
近 10 日涨幅
近 20 日涨幅
成交量相对 20 日均量
成交额相对 20 日均额
日涨跌幅
近 15 日涨停次数
振幅变化
```

然后把这 20 天的特征拉平成一个向量。

可以理解成：

```text
20 日走势窗口 -> 技术形态指纹
```

最后用余弦相似度比较两个形态指纹是否接近。

余弦相似度更关心：

```text
结构方向是否相似
```

而不是：

```text
股价绝对值是否相同
```

所以一只 10 元股票和一只 50 元股票，只要均线结构、涨跌节奏、量能结构、相对高低点位置相似，也可能被判断为相似。

---

## 7. 为什么你肉眼看图觉得不像

你截图里用的是同花顺公司 PK 图，它比较的是：

```text
同一段日历时间内，两只股票的累计涨跌幅曲线
```

而当前代码比较的是：

```text
002179 指定区间提取出的模板窗口
vs
候选股票当前最新 20 日窗口
```

两者不完全一样。

当前代码不是在比较：

```text
002179 2026-04-20 ~ 2026-05-10
vs
天士力 2026-04-20 ~ 2026-05-10
```

而是在比较：

```text
002179 的模板窗口
vs
天士力当前最新 20 日窗口
```

并且它不是纯粹比较累计涨跌幅曲线，而是综合比较均线、量能、涨跌幅、相对高低点等特征。

所以会出现：

```text
代码认为相似度不低；
但你肉眼看同花顺 PK 图觉得不像。
```

这不是你理解错了，而是当前算法的相似度定义和你肉眼判断的“走势曲线相似”还不完全一致。

## 8. Top 20 结果怎么解读

假设输出：

```text
002265 建设工业: 平均67.0% 最大71.0% 匹配4次
```

意思是：

```text
002265 当前最新 20 日窗口
和 002179 的多个模板窗口进行了比较；
其中有 4 次相似度超过 threshold；
超过阈值的这些匹配平均相似度为 67.0%；
最高一次相似度为 71.0%。
```

### 8.1 平均相似度

```text
平均67.0%
```

表示所有超过阈值的匹配结果的平均分。

注意，它不是和全部模板窗口的平均值，而是和超过阈值的那些结果取平均。

### 8.2 最大相似度

```text
最大71.0%
```

表示这只股票当前窗口和某一个模板窗口的最高相似度。

这个值高，说明它至少和模板中的某一段很像。

### 8.3 匹配次数

```text
匹配4次
```

表示它和模板库中有 4 个模板窗口超过了阈值。

匹配次数越多，通常说明不是偶然局部相似，而是和模板区间的多个阶段都比较像。

---

## 9. 应该优先看哪些股票

不要只看平均相似度，也不要只看最大相似度。

建议优先看：

```text
最大相似度高
匹配次数多
平均相似度也不低
```

例如：

```text
最大相似度 >= 70%
匹配次数 >= 3
平均相似度 >= 62%
```

这种一般比只匹配 1 次的股票更值得关注。

### 9.1 相似度高但匹配次数少

比如：

```text
某股票 平均63.0% 最大63.0% 匹配1次
```

这种可能只是和某一个模板局部相似，稳定性不够。

### 9.2 匹配次数多但平均不高

比如：

```text
某股票 平均58.0% 最大65.0% 匹配8次
```

说明整体有类似结构，但强度不够，需要结合 K 线判断。

### 9.3 最大很高但平均一般

比如：

```text
某股票 平均60.0% 最大75.0% 匹配2次
```

说明某个局部很像，但整体不一定像。

## 10. 参数怎么调

### 10.1 threshold

相似度阈值。

示例：

```bash
--threshold 0.55
```

含义：

```text
只有相似度 >= 55% 的匹配才保留。
```

建议：

```text
0.45：结果多，适合初筛，但噪声较多
0.55：平衡，当前推荐值
0.60：更严格，结果更少
0.65：很严格，容易漏掉潜在股票
```

如果你感觉结果太少：

```bash
--threshold 0.45
```

如果你感觉结果太杂：

```bash
--threshold 0.60
```

---

### 10.2 top-k

输出前多少只股票。

示例：

```bash
--top-k 20
```

如果你想多看一些：

```bash
--top-k 50
```

---

### 10.3 recent-windows

候选股票取最近几个窗口。

示例：

```bash
--recent-windows 1
```

推荐先用：

```text
--recent-windows 1
```

这样速度快，结果更聚焦当前最新形态。

如果你想增加容错：

```bash
--recent-windows 3
```

但会更慢，也可能增加一些不那么当前的形态。

---

### 10.4 skip-backtest

跳过历史回测。

示例：

```bash
--skip-backtest
```

建议你平时初筛都加上它。

因为历史回测很慢，而你盘后快速找票时，先看相似股票就够了。

推荐命令：

```bash
python cli/ml_match.py --code 002179 --date-start 2026-04-20 --date-end 2026-05-10 --threshold 0.55 --top-k 20 --recent-windows 1 --skip-backtest --workers 8
```

---

### 10.5 workers

并发线程数。

示例：

```bash
--workers 8
```

建议：

```text
4：比较稳
8：一般推荐
12 或更高：不一定更快，可能受硬盘 IO 限制
```

如果你发现开 8 和开 4 差不多，说明瓶颈主要在磁盘读取或 pandas 指标计算，不是线程数。

## 11. 为什么 180 行快速尾部读取会更快

你当前 Step 2 是扫描候选股票的当前最近形态。

既然只看最近 20 日窗口，就没必要把每只股票几年的历史数据全部读进来。

所以优化后采用：

```text
快速尾部读取最近约 180 行
```

因为日线 CSV 一行对应一个交易日，所以：

```text
180 行 ≈ 最近 180 个交易日
```

大概是 8 到 9 个月。

为什么不是只读 20 行？

因为指标需要预热，比如：

```text
SMA60
过去60日高点
过去60日低点
20日均量
15日涨停次数
SMA60_5日前
```

所以必须多读一些数据。180 行比较安全。

## 12. 当前最推荐的操作流程

### 第一步：先用手动模板快速初筛

```bash
python cli/ml_match.py --code 002179 --date-start 2026-04-20 --date-end 2026-05-10 --threshold 0.55 --top-k 20 --recent-windows 1 --skip-backtest --workers 8
```

看输出 Top 20。

---

### 第二步：人工过滤肉眼明显不像的票

重点看：

```text
1. 最大相似度是否高
2. 匹配次数是否多
3. 当前K线是不是低位启动而不是高位加速
4. 是否已经涨停或连续大涨
5. 是否靠近前高压力
6. 量能是否健康
```

---

### 第三步：降低或提高阈值复查

如果结果太少：

```bash
python cli/ml_match.py --code 002179 --date-start 2026-04-20 --date-end 2026-05-10 --threshold 0.45 --top-k 30 --recent-windows 1 --skip-backtest --workers 8
```

如果结果太杂：

```bash
python cli/ml_match.py --code 002179 --date-start 2026-04-20 --date-end 2026-05-10 --threshold 0.60 --top-k 20 --recent-windows 1 --skip-backtest --workers 8
```

---

### 第四步：对重点票再看分钟级别买点

相似度匹配只是初筛，不建议直接买。

更合理的顺序是：

```text
形态相似度初筛
-> 日线策略确认
-> 题材或行业共振
-> 资金流确认
-> 分钟级别买点确认
```

## 13. 自动模板模式怎么操作

如果你不想指定日期，只想让代码自动从股票里学习模板：

```bash
python cli/ml_similarity.py --template 002179 --template-mode auto --threshold 0.55 --top-k 20 --recent-windows 1 --workers 8
```

多只股票：

```bash
python cli/ml_similarity.py --template 002179,600535,002265 --template-mode auto --threshold 0.55 --top-k 30 --recent-windows 1 --workers 8
```

### 13.1 template-mode 参数

常用值：

```text
auto：自动判断启动点，推荐
prelaunch：偏启动前窗口
launch：偏启动初期窗口
both：启动前 + 启动初期
recent：直接取最近窗口
all：取历史所有窗口，不太推荐
```

推荐：

```bash
--template-mode auto
```

如果你想找“还没启动但快启动”的形态，可以尝试：

```bash
--template-mode prelaunch
```

如果你想找“刚开始启动”的形态，可以尝试：

```bash
--template-mode launch
```

## 15. 当前工程存在的不足

当前版本已经能跑，并且速度已经通过快速尾部读取优化。

但还有几个不足：

### 15.1 相似度更偏技术结构，不是纯走势曲线

当前相似度主要看：

```text
均线结构
量能结构
涨跌幅结构
相对高低点位置
```

它不完全等同于你肉眼看到的同花顺 PK 图。

后续建议增加：

```text
20 日累计涨跌幅曲线相似度
```

让走势曲线分数占更高权重。

---

### 15.2 输出信息还不够直观

现在输出：

```text
代码 名称 平均相似度 最大相似度 匹配次数
```

建议后续增加：

```text
最新日期
候选窗口日期
最像的模板编号
最像的模板日期
走势分
结构分
近5日涨幅
近10日涨幅
近20日涨幅
距离60日高点%
距离60日低点%
量能比
```

这样你才能更快判断它到底是低位启动，还是高位加速。

---

### 15.3 回测仍然比较慢

如果不加：

```bash
--skip-backtest
```

历史回测仍然要扫描很多历史窗口，所以会慢。

建议日常筛选先跳过回测，只有要验证模板有效性时再跑回测。

---

## 16. 建议下一步优化方向

为了让结果更贴近你肉眼看到的“走势像不像”，建议下一版改成：

```text
最终相似度 = 走势曲线相似度 * 0.6
           + 日涨跌节奏相似度 * 0.2
           + 技术结构相似度 * 0.2
```

其中：

```text
走势曲线相似度：
比较 20 日累计涨跌幅曲线

日涨跌节奏相似度：
比较每天涨跌幅节奏

技术结构相似度：
比较均线、量能、相对高低点位置
```

这样输出会更接近你在图上看到的走势形态。

## 17. 常见报错和处理

### 17.1 unrecognized arguments: --skip-backtest

说明你的 `cli/ml_match.py` 还没替换成支持 `--skip-backtest` 的版本。

处理：

```text
替换新的 cli/ml_match.py
```

---

### 17.2 unrecognized arguments: --workers

说明对应入口脚本还没支持 `--workers`。

处理：

```text
如果是 ml_match.py 报错，替换 cli/ml_match.py
如果是 ml_similarity.py 报错，替换 cli/ml_similarity.py
同时确认 runner.py 已经替换
```

---

### 17.3 某些股票名称为空

比如输出：

```text
002100 : 平均62.2% 最大62.2% 匹配1次
```

说明这个股票的名称没有从 CSV 或映射里成功读取。

不影响相似度计算，但后续可以加一个名称兜底函数，从你的股票基础池或行情接口中补齐名称。

---

### 17.4 扫描很慢

先确认是否用了这些参数：

```bash
--recent-windows 1 --skip-backtest --workers 8
```

如果还是慢，说明本地 CSV 很多或磁盘 IO 慢。

可以考虑后续增加：

```text
cache/ml_features
```

把计算好的指标缓存起来，避免每次重新计算。

## 19. 最重要的使用原则

这套工具不是让你直接买入，而是帮你从 1000 多只股票里快速缩小范围。

正确定位是：

```text
形态相似度初筛工具
```

不是：

```text
自动买卖决策工具
```

最终还要结合：

```text
日线位置
量能
题材
资金流
分时走势
分钟级别买点
大盘环境
```

如果只看相似度，很容易选到“技术结构像，但实际走势不理想”的股票。

## 20. 一句话总结

当前工程的核心用法是：

```text
你给一只或几只已经走出来的强势股；
代码从它们历史走势里提取 20 日形态模板；
然后扫描 cache/hist 股票池里每只股票当前最近 20 日形态；
找出技术形态最像的股票；
作为后续人工复盘和分钟买点确认的候选池。
```

当前最推荐命令是：

```bash
python cli/ml_match.py --code 002179 --date-start 2026-04-20 --date-end 2026-05-10 --threshold 0.55 --top-k 20 --recent-windows 1 --skip-backtest --workers 8
```
