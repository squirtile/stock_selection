# ML 形态匹配引擎 — 使用指南

## 这个模块干什么？

一句话：**你给一只"牛股"的某个时间段，它在全市场找出当前走势最像它的股票，并告诉你这些股票历史上出现类似形态后涨了没有。**

---

## 两条路，选哪条？

```
你有一只看好的牛股，知道它什么时候开始涨的
    │
    ├── 是 → 用「路线A: 单股票形态匹配」(推荐, 最常用)
    │
    └── 否, 我有多只好股票想批量处理 → 用「路线B: ML模型流水线」
```

**大部分情况用路线A就够了。**

---

## 路线A: 单股票形态匹配 (推荐)

### 你要准备什么？

3样东西：
- **股票代码**：比如 `000009`
- **日期区间**：这只股票大涨**之前**或大涨**初期**的一段日期，比如 `2024-03-01` ~ `2024-03-20`
- 这只股票必须在 `cache/hist/` 里有数据

### 它做了什么？

```
你给: 000009 在 2024-03-01~03-20 的走势 (你认为这是"启动形态")

系统做:
  Step 1 ─ 把这20天的走势转成数字特征（均线、量能、振幅等14个指标）
  Step 2 ─ 拿这个特征去和全市场1155只股票"最近20天"的特征比，算相似度
  Step 3 ─ 排除相似度低于60%的，剩下的按相似度排名 → Top 20
  Step 4 ─ 对这20只股票做回测：历史上它们出现类似形态后，涨了没有？
```

### 怎么运行？

```bash
# CLI 方式
python cli/ml_match.py --code 000009 --date-start 2024-03-01 --date-end 2024-03-20
```

```python
# Python 方式
from ml_engine.runner import match_single_stock_pattern

result = match_single_stock_pattern(
    template_code='000009',           # 股票代码
    date_start='2024-03-01',          # 启动形态开始日期
    date_end='2024-03-20',            # 启动形态结束日期
    similarity_threshold=0.60,        # 相似度门槛 (0.60=60%)
    top_k=20,                         # 返回前20只
    hold_days_list=[1, 3, 5, 10],    # 回测: 持有1/3/5/10天
)

# 看结果
print(result['similarity_df'])        # 相似度排名表
print(result['backtest_summary'])     # 回测汇总表
```

### 结果长什么样？

**相似度排名**：哪些股票当前走势最像你的模板

| 代码 | 名称 | 平均相似度% | 最大相似度% | 匹配次数 | 最新匹配日期 |
|------|------|------------|------------|---------|-------------|
| 600415 | 小商品城 | 76.8 | 77.7 | 4 | 2026-05-21 |
| 000559 | 万向钱潮 | 75.8 | 76.0 | 2 | 2026-05-21 |
| ... | ... | ... | ... | ... | ... |

**回测汇总**：这些股票历史上出类似形态后，表现如何

| 持有天数 | 信号次数 | 胜率% | 平均收益率% | 盈亏比 |
|---------|---------|-------|-----------|-------|
| 3 | 953 | 48.06 | 0.31 | 1.28 |
| 5 | 933 | 47.48 | 0.60 | 1.42 |
| 10 | 898 | 46.21 | 1.15 | 1.35 |

> 盈亏比 > 1 说明平均盈利大于平均亏损，策略整体有正期望。

### 参数怎么调？

| 参数 | 默认值 | 什么意思 | 怎么调 |
|------|--------|---------|--------|
| `similarity_threshold` | 0.60 | 相似度门槛 | 结果太少就降到0.50，结果太多就升到0.70 |
| `top_k` | 20 | 返回几只 | 看Top 20够了 |
| `hold_days_list` | [1,3,5,10] | 回测持有几天 | 短线看1-5天，中长线加10、20 |
| `recent_windows` | 3 | 候选股取最近几个20天窗口 | 默认3就行 |

### 输出文件在哪？

`output/ml_similarity/pattern_match_<股票代码>_<时间戳>.xlsx`

打开有4个sheet：相似度排名、明细匹配、回测明细、回测汇总。

## 路线B: ML模型训练流水线

### 什么时候用？

当你有多只"好股票"、不想指定具体日期区间、想让系统自动学习"什么样的形态后面会涨"时。

### 它做了什么？

```
你给: 多只牛股代码列表, 比如 ['000009','000027','000039']

系统做:
  Step 1 ─ 提取这些股票的所有历史窗口, 自动标注:
            未来5天涨了 ≥5% → "好形态"
            未来5天没涨到5% → "坏形态"
  Step 2 ─ · RandomForest 分类器
            → 模型学会判断: "给定一个形态, 后面涨5%+的概率是多少"
  Step 3 ─ 用余弦相似度在全市场找形态相似的股票
  Step 4 ─ 对相似股票用ML模型做回测
```

### 怎么运行？

```bash
# 指定模板股票
python cli/ml_train.py --template 000009,000027,000039 --full-pipeline

# 让系统自动选近期涨最好的20只
python cli/ml_train.py --auto-template --auto-top-n 20 --full-pipeline
```

```python
from ml_engine.runner import run_ml_pipeline

result = run_ml_pipeline(
    template_codes=['000009','000027','000039','000050','000058'],
    similarity_threshold=0.60,
    hold_days_list=[1, 3, 5, 10],
    top_k=50,
)
```

### 如果你想分步跑

```bash
# 第一步: 只训练模型
python cli/ml_train.py --template 000009,000027

# 第二步: 只扫相似度 (不训练)
python cli/ml_similarity.py --template 000009,000027 --threshold 0.60 --top-k 50

# 第三步: 只回测 (用训练好的模型)
python cli/ml_backtest.py --model output/ml_models/ml_pattern_model.pkl --hold-days 3,5,10
```

## 路线A 和 路线B 的区别

| | 路线A (单股票匹配) | 路线B (ML模型流水线) |
|---|---|---|
| **你需要给什么** | 1只股票 + 精确日期 | 多只股票代码 |
| **匹配方式** | 余弦相似度 | 余弦相似度 + ML模型打分 |
| **找什么** | 当前处于类似启动阶段的股票 | 任何历史形态相似的股票 |
| **回测** | 相似度回测 | ML模型回测 |
| **速度** | 2-3分钟 | 3-5分钟 |
| **适用场景** | "这只票3月启动前长这样, 现在谁长这样?" | "我这5只票都不错, 全市场还有谁?" |

## 常见问题

**Q: 为什么我的股票代码跑不了？**
A: 必须先跑过日线数据下载，股票才有缓存。检查 `cache/hist/` 目录下有没有 `<代码>_bs.csv`。

**Q: 相似度阈值设多少合适？**
A: 0.60 是中等门槛（默认），0.70 偏严格，0.80 很严格（可能找不到结果）。找不到结果就降阈值。

**Q: 回测的胜率才48%，是不是没用？**
A: 看盈亏比，不看胜率。盈亏比 1.4 意味着赢一次赚的钱能覆盖输1.4次。48%胜率 + 盈亏比1.4 ≈ 正期望。

**Q: 可以跑创业板/科创板吗？**
A: 不能。300/301/688 开头的股票被基础筛选过滤了，缓存里没有。

**Q: 需要联网吗？**
A: 不需要，所有数据都来自本地 `cache/hist/` 缓存。
